### Hi, I'm Jacob 👋

Software engineer focused on **operational, hardware, and integration-heavy systems** — the
software businesses actually run on. Four years building and owning production applications across
web, mobile, rugged Android devices, and data pipelines.

I tend to work end-to-end: gather the requirements, design the system, ship it, and keep it
running. I care about building things that hold up — hardening for failure modes before they
happen, not patching after.

**Most of my work is private** — production software for businesses and clients — so this profile
is light on repos by nature, not by output. A couple of public samples that show how I think:

- **[claude-overlay-electron](https://github.com/jhaqq/claude-overlay-electron)** — a desktop
  overlay for Claude Code I built and use daily: multi-tab sessions, a live review pane, and
  transcript tailing. Electron + Svelte 5, node-pty + xterm.js, tested with Vitest.

- **[conduit-spec-compiler](https://github.com/jhaqq/conduit-spec-compiler)** — an experiment in
  treating a software specification as a queryable relational database, so completeness becomes a
  SQL query. Compiles validated specs into code through verified LLM dispatch. Shows how I
  approach system architecture (and how I assess my own work honestly).

- **[samsung-remote](https://github.com/jhaqq/samsung-remote)** — a self-hosted PWA remote for my
  Samsung TV when I didn't have a physical one. React + Fastify + SmartThings, Dockerized. A small,
  complete full-stack product, end to end.
